```text
计划生成-代码 vs ToolCall


            Code-as-Action vs ToolCall 核心对比

  对比维度            ToolCall         Code-as-Action      结论

可固化性     每次都要调 LLM           验证通过 → 固化为代码            Code 胜出
         无法沉淀为资产             CI 零 Token 回归

调试能力     黑盒调用                生成标准代码                  Code 胜出
         只能看日志排查             IDE 单步调试 + Review

表达力      受限于预定义工具集           代码表达无上限                 Code 胜出
         无法组合复杂逻辑            支持自定义断言

执行成本     每次回归都消耗 Token       固化后零 Token 执行           Code 胜出
         每做一个行为就是一次模型请求      一次能生成 N 个行为动作，可编程

遵循性      训练到模型内              依赖模型的 coding 能力        ToolCall 胜出
```

```text
    总结

•   Code-as-Action：代码生成比 ToolCall 表达能力更好、容易变更、也更经济

•   脚本固化：日常回归走工程脚本，AI 只在生成/自愈时介入，但整体一套方案
```
