# Research Overview

## Research Topic

《知识工程驱动的 AI 实践》60-90 分钟中文演讲。听众包括技术团队内部成员和客户技术听众。

## Core Research Questions

1. 为什么强模型时代仍然需要工程化与 SDD，而不能只依赖提示词技巧或工具使用。
2. 软件工程基本功如何迁移到 AI 文件工程：SoC、抽象/面向对象、文件目录结构、接口契约、测试验证。
3. MindFlow 如何通过规约、方向、模块化、解耦、状态、能力和 Reflection 表达自我演进思想。
4. POMASA / MAS 为什么应被理解为强状态编排系统，而不是多个 Agent 聊天。
5. 如何解释状态控制三分法：强约束固定、有建议的 AI 自主判断、完全 AI 控制，以及灵活性与可靠性的反向关系。
6. Skill、工程文件和 Rules 如何作为可执行知识共同支撑 AI 工作系统。
7. Knowledge 与 Story 如何互相转化，为什么可以用 Flyway Migration 类比 Story 到 Knowledge 的演进。
8. BA / DEV / QA / UI / Agent 如何作为“思考帽”围绕同一仓库和强状态空间协作。
9. 如何构建贯穿输入到输出的统一流程：需求进入，经过 Tasking、澄清、设计、计划、实现、Review、测试和验收，最后代码出去，并且验收测试通过。
10. 如何把 Agent 设计、提速、测试评估、编排、工具层改造、长时间运行和无人值守放回统一流程中讲清楚。

## Research Scope

### Includes

- 软件工程基本功与 AI 文件工程。
- MindFlow、POMASA、MAS、Skill、Rules、强状态、分层 Plan、Agent 编排。
- 知识工程、Story、Knowledge、AC、ADR、工程文档、Capability、测试、eval case、workflow pattern。
- BA / DEV / QA / UI / Agent 的单仓库协作流程。
- 自动化、长时间运行、低人工干预、Agent Teams 展望。
- Harness Engineering、Agent Eval、CLI for Agents、Agent 可观测、Agent Runtime 等参考材料。

### Excludes

- 工具入门教程。
- 产品营销稿。
- 无来源支撑的行业泛论。
- DOCX、PDF 或单独 slide 提纲交付物。

## Research Stance

本项目以工程化、可验证、可落地为立场。演讲要有判断力，但不得夸大工具能力。外部事实、案例、术语和观点必须可追溯。用户原创观点应被保留，并在研究材料中标注为用户输入。

## Final Deliverables

最终只输出两份文件：

1. `_output/final-speech.md`：完整中文演讲稿，可直接照稿讲。
2. `_output/presentation.html`：用于现场演示的 HTML 文件，类似 PPT，一屏一主题，适合投屏。
