# Output Template

## Final Title

《知识工程驱动的 AI 实践》

## Final Deliverables

Only two final files should be produced:

1. `_output/final-speech.md` — complete Chinese speech manuscript for a 60-90 minute talk.
2. `_output/presentation.html` — lightweight HTML presentation for live projection, one screen per topic.

Do not produce DOCX, PDF, or a separate slide-outline file as final deliverables.

## Markdown Speech Manuscript Requirements

The manuscript must:

- Use natural spoken Chinese.
- Be professional but not obscure.
- Keep language clean, simple, and concise.
- Show engineering judgment.
- Avoid marketing tone and anxiety-driven language.
- Avoid frequent “不是……而是……” sentence patterns.
- Use paragraphs as the main form, not dense bullet lists.
- Include necessary citations and a reference list.
- Include appendix or Q&A material only when useful for the speaker.

## Required Main Structure

1. 为什么还要工程化与 SDD。
2. 软件基本功：SoC、抽象/面向对象、文件目录结构。
3. MindFlow 的自我演进。
4. ProMaster / POMASA 与 MAS 的强状态。
5. 状态控制三分法。
6. Skill、工程文件与 Rules。
7. Knowledge 与 Story。
8. Story 到 Knowledge 的 Flyway 类比。
9. Skill 与 Story 的验收。
10. 角色而非人。
11. 统一流程与单线程协作。
12. 多线程协作流程。
13. 自动化与无人值守。
14. Agent 设计、提速、测试评估、AI 编排、工具层改造、展望。
15. 附录/Q&A。

## HTML Presentation Requirements

The HTML presentation must:

- Be a small, portable HTML file.
- Work without complex external dependencies.
- Use one screen per topic.
- Keep each screen readable on a projector.
- Prefer short phrases, diagrams, and structured visual rhythm.
- Match the manuscript's chapter order.
- Include source notes only where they help the live talk.

## Citation Format

Use a readable citation style:

- In the manuscript body, cite sources naturally when needed.
- In the reference list, include source title and URL or repository-relative path.
- For user original ideas, identify them as user input or project framing in research notes rather than pretending they are external sources.
