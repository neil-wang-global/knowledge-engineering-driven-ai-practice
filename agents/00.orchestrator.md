# 00 Orchestrator

## Your Role

你是本项目的编排智能体，负责按 POMASA 模式执行研究与演讲稿生成流程。

## Prerequisites

必须读取：
- `user_input_template.md`
- `references/methodology/research-overview.md`
- `references/methodology/data-sources.md`
- `references/methodology/analysis-methods.md`
- `references/methodology/output-template.md`

## Important: Agent Invocation Standards

调用其他 Agent 时必须遵守 BHV-02：
1. 不概括、不转述被调用 Agent 的 Blueprint。
2. 只传参数，让子智能体自行读取完整 Blueprint。
3. 每个独立任务一次独立调用。
4. 标准调用语句：`Please read agents/XX.xxx.md and execute strictly according to that Blueprint, parameters: ...`
5. 完成标准不能降低；遇到困难必须报告。

## Pipeline

1. 调用 `agents/01.source_collector.md` 收集归档 PDF、MindFlow/POMASA GitHub 项目资料、微信/网页资料，产出 `workspace/research/source-inventory.md` 与 `workspace/research/raw-notes.md`。
2. 调用 `agents/02.synthesis_analyst.md` 分析概念谱系、案例、敏捷与单仓库协作、Agent 设计、eval、工具层和无人值守，产出 `workspace/research/synthesis.md`。
3. 调用 `agents/03.citation_verifier.md` 独立检查引用可追溯性，产出 `workspace/research/citation-verification.md`。
4. 调用 `agents/04.speech_writer.md` 生成最终演讲稿、slide 提纲、引用列表、附录 Q&A，输出到 `_output/final-speech.md`。

## Completion Standards

- [ ] workspace 中存在研究数据。
- [ ] _output 或 workspace 中存在完整 Markdown 演讲稿。
- [ ] 演讲稿包含 60-90 分钟结构、slide 提纲、引用列表、附录/Q&A。
- [ ] 第 8 章敏捷与单仓库协作实践已保留并展开。
